## 1. 产品一句话

iwalk.pro 是 Walker 的个人实践场：它把文章、点子、资源、学习轨道、项目、需求匹配和站主后台放在同一个系统里。普通用户可以阅读、搜索、探索资源；受邀用户可以使用“AI 帮你选”；站主可以在后台复盘需求、管理内容、生成选题和观察命中率。

## 2. 用户身份

| 身份 | 能做什么 | 看到的状态 |
| --- | --- | --- |
| 访客 | 浏览公开内容、搜索、查看文章/资源/点子/项目/关于页 | “AI 帮你选”显示“受邀可解锁” |
| 受邀用户 | 输入邀请码后使用需求匹配器，让 AI 帮自己收窄工具、内容和下一步行动 | 匹配器可打开聊天面板 |
| 管理员 | 登录后台，管理内容、查看需求洞察、选题池、复盘、命中率和 AI Gateway | 后台页面显示 Admin 导航和登出按钮 |

## 3. 全站通用入口

### 侧边导航

桌面端左侧有固定导航。主要入口包括：

- 首页：回到 Walker 画布首页。
- 搜索：打开快捷搜索面板。
- 内容：进入内容宇宙。
- 关于：了解 Walker、站点定位和公开数据。
- 项目：查看 Ferry 等项目。
- 学习：查看学习轨道、思考和学习指南。
- 资源：查看 AI 资源，并进入“AI 帮你选”。
- 点子：查看点子卡牌。

点击任一导航项会跳转到对应页面。移动端会收窄布局，核心入口仍保留。

### 快捷搜索

入口：

- 点击侧边栏“搜索”。
- 或使用 `Ctrl + K`。

打开后会出现搜索框。输入关键词后，系统会在公开内容和页面入口中筛选结果。点击结果会跳到对应页面；点击背景或按 `Esc` 会关闭面板。找不到内容时，会显示空状态，并保留“快捷跳转”按钮。

### 主题按钮

首页和导航区域提供主题切换入口。点击后会切换站点视觉主题，用于改变浏览氛围，不影响内容和数据。

## 4. 首页

路径：`/`

首页是 Walker 的入口画布，适合第一次进入时快速理解这个站。

主要区域：

- Walker 身份卡：展示站点名、当前标语和推荐/最近内容。
- 快速入口：文章、资源、点子、项目。
- 最近文章：直接进入最近公开文章。
- 点子火花卡：展示点子或临时灵感。

主要操作：

- 点击推荐/最近内容：进入对应文章详情。
- 点击“文章 / 资源 / 点子 / 项目”：进入对应页面。
- 点击主题：切换视觉主题。
- 拖动画布卡片：改变首页卡片位置，只影响当前浏览体验。

为什么有用：

首页不是普通介绍页，而是把“我是谁、最近在想什么、你能从哪里开始探索”放在一起，降低第一次访问的选择成本。

## 5. 内容宇宙

路径：`/content`

内容宇宙把公开内容按空间重新组织，而不是只按文章列表展示。

主要空间：

- 全部内容：显示所有公开内容。
- 前进系统：偏方法、系统、实践的内容。
- 生活现场：生活记录和现场感内容。
- 学习笔记：学习过程和思考。
- 工具资源：资源、工具、渠道。
- 作品项目：项目和已完成作品。
- 点子火花：想法、构思和可继续推进的点子。

主要操作：

- 点击空间筛选卡：页面只显示该空间内容，并更新“当前显示”说明和数量。
- 点击内容卡片：进入对应文章、项目或资源页面。
- 空间没有内容时：显示空状态，提示该空间暂无公开内容。

为什么有用：

用户不用先知道文章分类，也能按“我想看方法 / 生活 / 工具 / 点子”找到入口。

## 6. 文章与思考

路径：

```text
/posts
/posts/[slug]
/learn?tab=thoughts
```

文章列表用于集中阅读 Walker 的公开思考。

主要操作：

- 点击标签筛选：只显示包含该标签的文章。
- 点击文章标题：进入文章详情。
- 在文章详情中阅读正文、目录和上下篇导航。
- 在关于页或文章页的点赞按钮上点击：为该页面增加一次公开点赞计数。

为什么有用：

文章页保留完整思考，标签和搜索负责帮助用户快速定位主题。

## 7. 点子页

路径：`/ideas`

点子页用卡牌桌展示想法。点子有状态，例如构思中、进行中、已完成。

主要操作：

- 点击“全部 / 进行中 / 已完成”：筛选点子状态。
- 点击构思中的卡牌：先解除模糊遮罩。
- 再点击卡牌或“查看详细构想”：卡牌翻到背面，显示更完整的点子说明。
- 点击“返回牌面”：卡牌翻回正面。
- 点击“阅读全文”：进入该点子的完整文章页。
- 点击“加入点子共促”：跳转到资源页的社群信息区域。

为什么有用：

点子页让用户看到哪些想法还在构思、哪些已经验证或完成，也给潜在同行者一个可讨论的入口。

## 8. 资源页与“AI 帮你选”

路径：`/tools`

资源页展示 Walker 实际在用的 AI 信息源、工具、Skill、基础设施和推荐博主。页面顶部还有需求匹配入口“AI 帮你选”。

### 资源区

主要操作：

- 点击顶部章节导航：平滑滚动到信息源、AI 工具、Skill、基础设施、博主推荐。
- 扫二维码：加入对应社群或资源渠道。
- 点击外部链接：打开对应工具、Skill 或博主主页。

### AI 帮你选：访客态

访客点击“AI 帮你选”后，会先打开邀请码弹窗。

弹窗包含：

- 邀请码输入框。
- “你主要是什么人”可选输入。
- 学生、上班族、自由职业、想搞副业等快速身份按钮。
- “同意被 Walker 记住需求，用于选题改进”勾选项。
- 进入按钮。

点击“进入”后的结果：

- 邀请码为空：提示请输入邀请码。
- 邀请码太短：提示格式不正确。
- 邀请码错误：提示邀请码无效。
- 身份锚点包含手机号/邮箱等敏感信息：提示重新填写。
- 邀请码正确：弹窗关闭，进入受邀用户状态，并打开聊天面板。

### AI 帮你选：受邀/管理员态

聊天面板包含：

- 问题槽：最多保留最近 8 轮问答。
- 当前角色：来自身份锚点、背景选择或管理员状态。
- 当前卡点：AI 根据问题诊断。
- 快捷问题：改已有项目、写公众号、做网站、做报名表、学 AI、提效、数据分析。
- 输入框：描述自己的需求。
- 发送按钮：提交需求。
- 匿名加入选题库：默认勾选，可取消。
- 补充背景：可选择使用场景和 AI 阶段。
- 历史按钮：查看本机保存过的对话历史。
- 关闭按钮：关闭面板。

发送问题后的结果：

- 如果还未受邀，后端返回 401，前端重新打开邀请码弹窗。
- 如果问题需要继续诊断，AI 会给几个方向按钮让用户继续选择。
- 如果可以推荐，AI 会给主工具、备选工具、可复制提示词、下一步行动和站内资料。
- 如果不适合协助，AI 会给合规转向。
- 用户可以点击“做出第一版 / 知道下一步 / 方向不对 / 还缺教程”反馈结果。

为什么有用：

用户不需要先知道该用哪个 AI 工具，只要描述目标或卡点，系统会帮他把问题收窄成一个可执行动作；站主也能从匿名需求中发现内容缺口和选题。

## 9. 学习页

路径：`/learn`

学习页把“学习轨道、思考、帮你学”放在同一个页面。

页签：

- 轨道：展示不同学习层级、进度、里程碑、学习笔记、实践反思和参考资料。
- 思考：按年份和标签浏览学习相关思考。
- 帮你学：展示面向用户的学习指南入口。

主要操作：

- 点击“轨道 / 思考 / 帮你学”：切换页签，地址栏同步更新。
- 在轨道卡中点击学习笔记或参考资料：打开外部对话、文章或资料。
- 在思考页签点击标签：筛选文章。
- 在帮你学页签点击卡片：进入对应学习指南。

为什么有用：

它把“我学了什么、怎么学、你可以怎么跟着学”放在一起，既是记录，也是学习路径。

## 10. 项目页

路径：

```text
/projects
/projects/ferry
```

项目页展示 Walker 正在推进或已经沉淀的项目。

主要操作：

- 点击 Ferry 渡轮旗舰卡：进入 Ferry 项目详情。
- 点击子项：进入对应文章或外部链接。
- 查看状态标签：了解项目是建设中、已完成还是已归档。

为什么有用：

项目页把零散文章背后的长期工程和方法论收束起来，让用户知道哪些内容属于同一个方向。

## 11. 关于页

路径：`/about`

关于页说明 Walker 是谁、为什么建站、站点是什么、怎么做成，以及公开需求洞察。

主要区域：

- Hero：展示 Walker 的一句话定位。
- 关于我：身份叙事、引用、感谢、社交链接。
- 航行日志：个人路线时间轴。
- 为什么建：建站动机。
- 点子共促：加入交流的引导。
- 这个站是什么：内容、资源、点子、项目等入口说明。
- 怎么做的：AI 工具、成本、技术管线。
- 需求洞察：展示公开的聚合需求趋势。

主要操作：

- 点击“开始探索”：滚动到介绍区域。
- 点击“点子共促 / 了解更多”：跳转到资源页社群区域。
- 点击 GitHub / Bilibili / 小红书：打开外部主页。
- 点击 Email：复制邮箱，按钮短暂显示“已复制”。
- 点击点赞按钮：增加关于页点赞计数。
- 点击“回到点子共促”：滚动回 CTA 区域。

为什么有用：

关于页不是简历，而是解释这个站为什么存在、如何持续生长，以及用户为什么可以信任这套实践路径。

## 12. 管理员后台

后台路径从 `/admin/login` 进入。未登录访问 `/admin` 或其他后台页面时，会跳转到登录页。

### 登录页

路径：`/admin/login`

主要操作：

- 输入管理员密码。
- 点击“登录”：密码正确则进入后台；错误则显示“密码错误”。
- 点击“返回首页”：回到公开首页。

### 管理后台首页

路径：`/admin`

用途：

- 查看内容总数和主要类型数量。
- 从快捷操作进入内容管理、需求洞察、学习轨道。
- 查看最近内容并打开公开页。
- 点击登出退出管理员状态。

### 内容管理

路径：`/admin/content`

主要操作：

- 点击“新建内容”：进入内容编辑页。
- 在搜索框输入标题、slug 或标签：过滤内容表。
- 用类型筛选：只看 knowledge、tool、idea、project、learn 等类型。
- 用可见性筛选：只看 public、draft、private。
- 点击查看：打开公开内容；非公开内容显示“未公开”。
- 点击编辑：进入编辑页。
- 点击发布、草稿、私密：切换内容可见性，系统会弹确认框。
- 点击删除：系统会弹确认框，建议优先转草稿或私密。

### 选题池

路径：`/admin/topics`

用途：

把用户需求簇和站主灵感放在同一个池子里判断。

主要操作：

- 输入选题灵感并点击“加入选题池”：把站主灵感加入选题池。
- 点击“观察中 / 已采纳 / 已暂缓 / 已忽略 / 已创作”：切换选题状态列表。
- 点击“生成简报”：进入创作简报流程。
- 点击“采纳 / 暂缓 / 忽略”：改变选题处理状态。

### 需求复盘

路径：

```text
/admin/review?view=clusters
/admin/review?view=list
```

两种视图：

- 需求簇：从多个相似需求的角度看选题机会。
- 逐条复盘：逐条查看 NeedCase 证据。

主要操作：

- 点击“需求簇 / 逐条复盘”：切换复盘方式。
- 在逐条复盘中点击“待复盘 / 已采纳 / 已暂缓 / 已忽略”：切换状态列表。
- 对待复盘需求点击“采纳 / 暂缓 / 忽略”：记录站主判断。
- 点击“备注”：展开备注输入框，记录判断依据。

### 数据看板

路径：`/admin/insights`

用途：

查看需求数据、分类、趋势、用户反馈和内容缺口。它帮助站主判断“用户真实在问什么”。

主要操作：

- 点击顶部后台导航进入其他后台模块。
- 点击登出退出后台。

### 内容命中率

路径：`/admin/hit-rate`

用途：

按已发布内容观察关联需求的 resolved/stuck 结果，帮助站主判断内容是否真正解决了问题。

### AI Gateway

路径：`/admin/ai-gateway`

用途：

管理 AI Gateway 配置。

主要操作：

- 点击“保存配置”：保存当前配置。
- 点击“测试连接”：验证配置是否可用。
- 点击“撤销上次”：回到上一次配置。
- 点击“重置默认”：恢复默认配置。

## 13. AI 可读接口

当前已有公开 AI 读取入口：

```text
/llms.txt
/walker-style.md
/index.json
```

这些入口主要给 AI 或自动化工具读取，不是普通用户主要浏览路径。它们只应暴露公开内容，不应泄露 draft、private 或 admin-only 信息。

## 14. 后续更新方式

更新本文时按以下规则：

1. 先运行当前 App，不只看源码。
2. 用浏览器/CDP 打开新功能涉及的页面。
3. 截图保存到 `output/playwright/`。
4. 在对应页面段落追加新增按钮、状态和点击后的结果。
5. 如果新增页面，新增一个同级章节。
6. 在“版本记录”新增一条记录，写清 commit、截图证据、主要变化。
7. 不删除旧记录，除非功能已经明确下线；下线时标注“已下线”。

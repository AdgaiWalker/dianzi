# Content Intent

Different content has different standards. Do not judge all content by user demand.

## Intent Types

```text
self_record       # personal thought, life, resignation, road, reflection
public_answer     # tutorial, guide, answer to real user demand
tool_note         # tools Walker uses and has verified in practice
brand_signal      # who Walker is, what Walker believes, what Walker can help with
method_seed       # reusable experience not yet fully verified
skill_candidate   # bounded, callable, repeatedly effective method
```

## Rules

- `self_record`: personal thought does not need market validation.
- `public_answer`: tutorials and answers need real or likely user demand.
- `tool_note`: tools need Walker's own practice.
- `brand_signal`: brand content explains who Walker is and what Walker can help with.
- `method_seed`: methods need repeated reflection or repeated situations.
- `skill_candidate`: skills need domain boundary, input conditions, workflow, output form, and
  validation criteria.

## Two Creation Loops

Self-use loop:

```text
personal experience / thought / tool / life
-> private or draft record
-> review and organize
-> content, method, plan, or next action
-> update after practice
```

User-answer loop:

```text
real user asks / searches / needs
-> record demand event
-> match existing content, tool, method, or article
-> if no answer exists, create a topic candidate
-> publish answer when appropriate
-> collect feedback
```

## Public Boundary

Demand can trigger public answers, but it does not force private records to become public. Private
experience may be adapted, anonymized, summarized, or kept private.

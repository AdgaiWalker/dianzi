# NorthStar 社区边界

NorthStar 是多个个人系统之间的连接网络。它可以与 AdgaiWalker 个人系统并存，但不能把
iwalk.pro 直接改造成重型社区平台。

## 核心关系

```text
Personal System -> Publish Interface -> Community Network
```

- 个人系统负责沉淀一个人的内容、工具、经验、skill、项目和 Agent 上下文。
- 发布接口负责选择什么能进入社区，以及进入时的可见性、形态、去向和反馈方式。
- 社区网络负责让不同人的 idea、问题、项目、skill 和案例发生连接。

## NorthStar 应该承载什么

NorthStar 不只是信息流。它应该优先承载从个人系统发布出来的对象：

- idea：点子、方向、假设
- question：真实问题和需求
- answer：公开回答、教程、方案
- skill：可复用经验包
- project：项目和行动记录
- tool：工具和工作流
- case：案例和复盘

## 必须先有的接口证据

社区可以并存，但扩张前要有接口证据：

- 个人系统里有稳定的发布候选。
- 发布候选能转换成社区对象。
- 社区对象有清楚可见性、意图、状态和目标区域。
- 社区反馈能回流到个人系统。
- 发布不依赖泄露私人内容或管理员后台数据。

## 不要过早塞回个人系统

这些能力可以属于 NorthStar 社区本体，但不要默认塞进 iwalk.pro：

- 完整社区账号
- 社交信息流
- 关注/粉丝体系
- 公开想法市场
- 多人协作工作台
- 推荐算法
- 审核工作流
- 平台商业化

## 晋升规则

只有当一个模式在个人系统中有真实案例、重复发生、边界清楚、能被转换成社区发布对象时，
才把它推进 NorthStar。否则继续留在个人系统里验证。

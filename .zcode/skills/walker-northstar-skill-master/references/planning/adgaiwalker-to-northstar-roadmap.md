# AdgaiWalker 到 NorthStar 路线图

本文档负责说明 AdgaiWalker 个人系统、发布接口和 NorthStar 社区网络之间的关系。它不是
PRD，也不是 todo；它只回答：

```text
个人系统现在验证什么？
社区网络承载什么？
个人系统通过什么接口把内容发布到社区？
```

## 核心判断

```text
AdgaiWalker = 个人系统样板节点。
Publish Interface = 个人系统向社区开放的选择性发布接口。
NorthStar = 多个个人系统之间的连接网络。
```

个人系统和社区不是严格的先后关系。二者可以同时存在；当前的最小实验是先把 Walker 自己
这个个人系统节点跑清楚，再定义它如何向社区发布内容、想法、skill、项目和案例。

## 三条并行线

### 1. 个人系统线

目标：让 iwalk.pro 成为稳定的个人内容、工具、想法、生活记录、Agent 和 skill 沉淀入口。

当前重点：

- 内容能稳定发布、检索、展示。
- 管理端能支持内容维护和私密内容管理。
- Agent 能按问题读取内容、匹配答案和辅助创作。
- 经验能沉淀为 skill 候选或正式 skill。
- 公开内容和私密内容边界清楚。

这条线是当前主要执行线。

### 2. 发布接口线

目标：定义个人系统里的什么内容能进入社区，以及进入社区时的形态、权限、位置和反馈回流。

当前重点：

- 定义发布对象：`idea`、`question`、`answer`、`skill`、`project`、`tool`、`case`。
- 区分内部材料、发布候选、社区对象。
- 设计可见性、意图、状态、目标区域和反馈方式。
- 让社区反馈能回流到个人系统的需求池、选题、复盘或 skill 候选。

这条线是个人系统和社区之间的桥，详见
`references/personal-system-publish-interface-architecture.md`。

### 3. 社区网络线

目标：让多个个人系统之间可以通过 idea、问题、项目、skill 和案例发生连接。

当前边界：

- 社区可以已经存在或并行推进。
- 社区不等于把 Walker 的个人系统平台化。
- 社区本体可以有账号、空间、协作、审核和发现机制，但这些不能默认塞进 iwalk.pro。
- iwalk.pro 当前只负责证明一个个人节点和发布接口如何工作。

## 推荐推进顺序

推进顺序不是“社区最后才存在”，而是：

```text
个人节点先跑出稳定材料
-> 发布接口定义什么可公开、如何公开
-> 社区接收标准化发布对象
-> 社区反馈回流个人系统
-> 个人系统再更新内容、需求、复盘和 skill
```

社区可以并行存在，但当前开发不要跳过发布接口直接做大社区。

## 阶段信号

### Phase A：个人节点可用

证明信号：

- 内容、工具、想法、生活记录能稳定沉淀。
- 管理端能维护公开、草稿、私密、管理员内容。
- AI 能读取适合公开或授权读取的内容。
- 当前状态能由 `references/current-cycle-state.md` 和必要补充说明清楚。

### Phase B：个人 Agent 可用

证明信号：

- Agent 能按问题检索内容，而不是只列文章。
- Agent 能区分公开、私密、管理员内容。
- Agent 六模块边界逐渐清晰。
- 回答结果能被反馈和复盘。

### Phase C：需求到创作闭环可用

证明信号：

- 用户能表达身份、场景、问题和期待。
- 系统能沉淀需求事件，而不是只产生一次性对话。
- 管理端能复盘需求、匹配结果和反馈。
- 至少有一批内容由真实需求触发创作。

### Phase D：经验到 Skill 可用

证明信号：

- 原始经验、复盘、规律、skill 候选和正式 skill 能分层。
- 一个 skill 有清楚定义域、输入条件、输出形态和验证标准。
- Agent 能判断何时调用 skill、何时追问、何时拒绝或降级。
- 已有 skill 在真实任务中多次复现有效结果。

### Phase E：发布接口可用

证明信号：

- 个人系统中能标记发布候选。
- 发布候选能转换成社区发布对象。
- 发布对象能带上可见性、意图、状态、目标区域和反馈方式。
- 社区反馈能回流为需求、选题、复盘或 skill 候选。

### Phase F：社区网络扩展

证明信号：

- 多个个人系统能用相同接口发布对象。
- 社区能接收、展示、连接和反馈这些对象。
- 社区连接带来的价值高于单个个人网站。
- 有明确理由引入多人协作、发现机制、审核和推荐。

## 晋升规则

一个模式从 AdgaiWalker 个人节点进入 NorthStar 社区网络，必须同时满足：

```text
有真实案例
有重复发生
有可描述流程
有可验证结果
有明确边界
能被转换成社区发布对象
离开 Walker 本人后仍有部分价值
```

只要还停留在想象、愿景、单次成功或强依赖管理员手工判断，就继续留在个人系统中验证。

## 当前执行原则

```text
当前 todo 主要服务 AdgaiWalker 个人系统。
发布接口是个人系统与社区之间的桥，优先定义边界，不急着做复杂平台。
NorthStar 可以并存，但社区复杂度不能倒灌进 iwalk.pro。
任何社区化功能都必须先问：它属于个人节点、发布接口，还是社区网络本体？
```

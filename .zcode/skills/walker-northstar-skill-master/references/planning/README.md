# Planning Zone

规划区保存长期有效的方向、背景、需求分析、功能设计、架构设计、协议和边界。

规则：

- 规划区拥有边界权，不直接替代当前工作区的 PRD / plan / to-do list。
- 长期架构、协议、路线图、策略和不可越界事项属于这里。
- 规划材料只有在被当前 PRD / plan / to-do list 引用时，才参与当前执行。
- 当前阶段已经完成第一轮物理归位：规划材料集中放在本目录，当前执行三件套放在 `../working/`。

硬规则：

```text
规划区中即使文件名包含 prd / plan / todo，也只是背景、候选或历史设计材料。
它们没有当前执行权。
只有内容进入 ../working/prd.md、../working/plan.md、../working/to-do list.md 后，才拥有当前执行权。
```

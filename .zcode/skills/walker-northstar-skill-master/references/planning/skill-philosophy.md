# Skill Philosophy

Skill is not a prompt. Skill is a bounded, callable, verifiable package of repeatedly effective
experience.

## Definition

```text
Skill = experience function with a domain boundary

input inside domain -> useful output
input outside domain -> ask, refuse, route, or downgrade
```

## Admission Test

A skill candidate must have:

- domain: what it handles and what it does not handle
- input conditions: what information is required
- workflow: how it reasons or acts
- output form: what it returns
- validation criteria: how to know it worked
- failure boundary: when it should not pretend to solve the problem

## Maturity Path

```text
raw material
-> knowledge card
-> method seed
-> repeated practice
-> method card
-> skill candidate
-> registered skill
```

## Main File Role

`SKILL.md` should be the guardrail and resource router. It should not carry all background material.

Good main file:

- decides when to trigger
- names boundaries
- routes to references
- defines output rules
- refuses overreach

Bad main file:

- stores every article
- contains unbounded philosophy
- promises to solve all cases
- hides missing validation

# Human Gate Policy

本文档定义 `walker-northstar` 下 AI 协作开发的人类闸门。核心原则：

```text
AI 可以执行可撤回工作；不可逆、高成本、涉及权限和公开的动作必须由人确认。
```

## AI 可直接执行

满足以下条件时，AI 可以直接做：

- 读取文件、搜索文件、整理当前状态。
- 新增或修改普通 Markdown 文档。
- 小范围代码修改。
- 运行本地检查、测试、构建。
- 创建临时目录和临时克隆仓库。
- 同步已经明确授权的文档到对应仓库。
- 修正路径引用、拼写、格式、死链接。

前提是：动作可撤回、范围明确、不公开私密内容、不删除重要数据。

## 需要一句话确认

这些动作需要先说明影响，再获得用户确认：

- 移动一批文件。
- 重命名当前有效文档。
- 修改权限、认证、可见性策略。
- 修改数据结构、内容 schema、API contract。
- 在主项目中引入新依赖。
- 把当前文档归档。
- 打 tag、创建 release、推送远程。
- 改 `.gitignore`、子仓库、子模块、远程仓库关系。

## 高风险动作

这些动作必须给出 A / B 方案、风险说明和确认：

- 删除重要文件或目录。
- 批量重写历史文档。
- 公开、迁移或复制私密内容。
- 改生产环境配置。
- 改管理员权限或登录认证安全逻辑。
- 强制推送、reset、rebase 共享分支。
- 清理数据库、Redis、外部服务数据。
- 把个人系统内容自动同步到社区。

## 禁止自动执行

除非用户明确要求并确认，AI 不得自动：

- 绕过权限限制。
- 泄露管理员密码、token、cookie、密钥。
- 把 private/admin-only 内容发布到公开页面或社区。
- 为了通过检查而删除失败证据。
- 覆盖用户未提交的无关改动。
- 把未验证想法写成强制规则。

## 仓库闸门

```text
AdgaiWalker 产品仓库：改 reality，默认可执行可撤回动作。
walker-northstar-skill 仓库：改 blueprint/.ferry，默认可维护规则和文档。
NorthStar 仓库：改社区网络本体，除非用户明确切换，否则不进入。
```

跨仓操作时，必须说明：

```text
要改哪个仓库？
为什么不是另一个仓库？
是否需要同步或回写？
是否涉及推送远程？
```

## 记录要求

触发中高风险闸门后，应在 `execution-log-current.md` 中记录：

- 发生了什么。
- 为什么需要人类确认。
- 最终选择是什么。
- 这是否应沉淀为新规则。

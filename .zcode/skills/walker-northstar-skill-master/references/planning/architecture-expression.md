# Architecture Expression

Use this reference when the user asks for a big-picture explanation, architecture diagram, visual
report, PRD framing, or a way to explain why AdgaiWalker / NorthStar matters.

## Layered Diagram

```text
era layer
  AI execution capability spreads; judgment becomes scarce.
  ↓
protocol layer
  FerrySpec: human sovereignty, entropy reduction, blueprint/reality/log, Skill/Flow/Log.
  ↓
value constitution layer
  People are the purpose; AI is the tool. Data shows phenomena, not purpose.
  ↓
idea-driving layer
  Ideas connect people and AI, people and people, content and projects.
  ↓
content and experience kernel
  Thought / Tool / Life / Idea / Event / Answer / Method / Skill.
  ↓
human I/O                         AI I/O
  pages / cards / search / feedback    schema / index / API / MCP / Skill
  ↓                                  ↓
feedback calibration
  questions / search / likes / comments / outcomes / reviews.
  ↓
method evolution
  new rules / new methods / new skills / new boundaries / new content models.
```

## Explanation Pattern

When explaining the system, use this order:

```text
Why this exists
-> What contradiction it answers
-> What system boundary it has
-> What loop it strengthens
-> What proof would validate it
```

## Keep

- layered architecture
- personal node / publish interface / community network relation
- value constitution
- human I/O vs AI I/O
- feedback-to-method evolution

## Avoid

- treating an old architecture diagram as current implementation truth
- building NorthStar complexity inside AdgaiWalker
- turning visual architecture into a task list

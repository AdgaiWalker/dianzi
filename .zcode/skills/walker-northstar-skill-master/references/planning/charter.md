# 项目章程

## 核心判断

```text
AdgaiWalker = 个人系统样板节点。
Publish Interface = 个人系统向社区开放的选择性发布接口。
NorthStar = 多个个人系统之间的 idea 连接网络。
```

AdgaiWalker 是 Walker 当前的个人实践场。它验证一个人如何记录经验、用 AI 组织行动、发布
有用回答，并把反复有效的实践沉淀成方法或 skill。

NorthStar 可以作为社区网络并存。它不取代个人系统，而是连接多个个人系统发布出来的对象。

## 关系

```text
FerrySpec = 演化中的方法协议
AdgaiWalker / iwalk.pro = 个人系统样板节点
Publish Interface = 选择性发布桥
NorthStar = 社区网络
```

FerrySpec 提供方法框架。AdgaiWalker 在 Walker 的真实实践中测试这个框架。Publish
Interface 决定什么能从个人系统进入社区。NorthStar 连接这些被发布出来的 idea、问题、回答、
skill、项目、工具和案例。

## 价值宪法

- 人是目的，AI 是工具。
- 人拥有目标、判断、审美、发布、责任和不可逆决策。
- AI 帮助检索、组织、起草、比较、追问和执行可逆工作。
- 数据呈现现象，数据不设定目的。
- 公开方向，不公开脆弱。
- 系统帮助人成为自己，不把人变成系统对象。
- 重复劳动交给 AI，不可替代的经验留给人。
- 界面服务人的理解，结构服务 AI 的执行。

## 当前阶段

默认任务属于 AdgaiWalker 个人系统或 Publish Interface，除非用户明确说明任务是处理
NorthStar 社区网络本体。

短期目标：

```text
记录 Walker 的思考、工具、生活、需求、点子、内容、方法和实验
-> 用 Agent 帮助 Walker 找答案、做创作、推进点子
-> 让访客能从公开内容里找到答案
-> 只在有助于创作和验证时收集真实需求
-> 定义哪些对象可以选择性发布到社区
```

长期目标：

```text
多个个人系统发布经过选择的对象
-> 社区连接 idea、问题、回答、skill、项目、工具和案例
-> 反馈回流到个人系统
-> 方法和 skill 继续进化
```

## 边界

AdgaiWalker 不承担 NorthStar 的平台复杂度。

现在做：

- 个人采集
- 内容意图
- 答案匹配
- 邀请制身份理解
- 事件和反馈记录
- 方法与 skill 准入
- 发布候选和发布接口定义

默认留在 NorthStar 社区网络，不塞进 iwalk.pro：

- 完整社区账号
- 公开想法市场
- 协作网络
- 审核
- 推荐
- 平台商业化

## 判断规则

当请求模糊时，先判断它属于哪一层：

```text
个人系统节点
发布接口桥接
社区网络本体
```

默认边界句：

```text
AdgaiWalker 验证个人节点，Publish Interface 决定什么进入 NorthStar。
```

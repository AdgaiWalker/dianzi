## 1. 产品定位

> 盘根·AI 指南针是一个熵减器——把 AI 时代的混乱信息，通过实践验证，转化为有序的、可用的知识。

**三句话核心**：

1. **成为自己** — 盘根是一个人成为自己的路。校园站帮你站稳脚下，全球站帮你看见远方
2. **知识流动** — 想法在全球站被探索，实践验证后沉淀为校园站里可直接用的知识；生活中的问题也驱动新的探索
3. **推动 AI 大众化** — 帮助人们正确认识和使用 AI，实现个人价值与社会价值的统一

**底层驱动**：Ferry 框架（熵减思维：规划→执行→复盘）指导内容方向与质量保证。

## 9. 内容安全：敏感词管控

### 设计原则

- **两层防护**：输入拦截（不发 AI 请求）+ 输出过滤（AI 响应降级）
- **词库共享**：敏感词库位于 `@ns/shared`，两站共用同一份词库
- **降级策略**：命中敏感词时不报错，而是降级为本地关键词搜索（fallback），保证用户体验

### 敏感词分类

| 分类 | 示例方向 | 说明 |
|---|---|---|
| 政治 | 依法律法规动态调整 | 需配合监管要求实时更新 |
| 色情 | 色情、裸体、成人视频等 | 覆盖直接描述与常见变体 |
| 暴力 | 杀人、自制武器、自残等 | 含自杀方法类 |
| 赌博 | 赌博、博彩、百家乐等 | 覆盖线上赌场术语 |
| 违禁品 | 代开发票、买卖枪支、迷药等 | 含金融欺诈类 |
| 辱骂 | 傻逼、操你等 | 覆盖常见脏话 |

### 拦截流程

```
用户输入 query
  → checkSensitiveWords(query) 命中？
    → 是：降级为 fallback 结果（frontai）
    → 否：发送 AI 请求
      → AI 返回 summary
        → checkSensitiveWords(summary) 命中？
          → 是：降级为 fallback 关键词搜索（FallbackReason: sensitive_output）
          → 否：正常展示 AI 结果
```

### 技术实现

- **词库文件**：`packages/shared/src/sensitive.ts`
- **校验函数**：`checkSensitiveWords(text)` → `{ hit: boolean, words: string[] }`
- **FallbackReason 新增**：`sensitive_blocked`（输入拦截）、`sensitive_output`（输出过滤）
- **原型阶段**：客户端硬编码词库（~50 词），后续迁移至后端词库 + 增量更新

### 演进路线

| 阶段 | 词库管理 | 匹配算法 | 覆盖范围 |
|---|---|---|---|
| 原型（当前） | 客户端硬编码 | 字符串包含匹配 | 6 分类 ~50 词 |
| 后端搭建后 | 服务端词库 + 后台管理 | DFA 自动机 / 正则变体 | 持续扩充 |
| 正式上线 | 第三方内容安全 API | NLP 语义分析 | 全量覆盖 |
